Item 1 / Overall proposal

两阶段：Ego World 预测世界变化，IDM 把视觉变化变成机器人动作

腕部视角和固定/浮动机器人头部视角都算 ego view；关键是所有视角共享同一 context，并能接受实时外部 action condition。

Proposal one-liner: Tri-View builds an interactive Ego World + IDM foundation model. The Ego World predicts how the shared object-interaction-robot scene changes under external actions; the Image-Layered stage decomposes generated rollouts into robot, object-interaction, and scene layers so an IDM can decode pose/velocity/current-loop commands and a process reward can supervise further learning.

Stage 1 Ego World
Stage 2 IDM + layers
Core state object interaction
Control output pos / vel / current

1. How should world change?

给定当前 wrist / robot-head ego observations、共享 context 和外部 action condition，模型要预测 object、robot、scene 在未来如何变化，而不是只生成好看的多视角视频。

2. What interaction info?

Image-Layered 回答生成视频怎么用：robot layer 用于 IDM / pose / VO，object-interaction layer 用于过程奖励和 RL 自监督，scene layer 用于物理合理的增强和自适应。

3. How to command robot?

看得见臂时从 robot layer 做 pose / joint estimation；看不见臂时用 wrist/head VO、SLAM 或 visual geometry 估计 egomotion；最后由 IDM/control decoder 发 position、velocity 或 current-loop 命令。

Main references

主参考扩成三条主线 + 四个支撑模块

Gamma-World

multi-agent interactive world-model baseline。它证明多 agent / 多控制流同步生成这条线已经有人做；我们的差异要放在 ego views、object-interaction-robot layers 和 IDM/control decoding。

Qwen-Image-Layered

image-layer decomposition baseline。它说明 RGB 图像可以拆成 RGBA layers；我们的任务是把 layer 从可编辑图层升级成 robot / object-interaction / scene 训练信号。

ObjectForesight

object layer 的关键类比：它预测 object 未来 6-DoF trajectory，并用 segmentation / reconstruction / pose estimation 从人类视频构建伪 GT。我们可以把 object layer 当成交互过程奖励和 RL 监督。

EgoWM / LOME / EgoSim

支撑 Ego World：action-conditioned egocentric generation、shared 3D state update、human/object interaction video。它们把“如何让 ego video 随 action 改变”变成可写的阶段一目标。

EVA / RoboEngine / ORB-SLAM3 / VGGT

EVA 支撑 IDM 和 executability reward；RoboEngine 支撑 robot/object mask 后的物理任务感知背景替换；ORB-SLAM3 / VGGT 支撑看不见臂时的 VO、SLAM 和 feed-forward visual geometry。

Rewritten story

Object-interaction-robot，而不是简单 tri-view 视频

新的 intro 已写回 dashboard 项目卡。这里保留周报版完整解释。

Ego views

腕部相机、固定机器人头部相机、浮动机器人头部相机都算 ego。三视角不是“必须 detached exo”，而是同步的 robot-centered POVs。

Shared context

所有视角视频要 share context：同一 scene、object、action、timestamp、task result，并尽量有 POV overlap。否则它只是多路独立生成。

External action

模型必须能接受实时外部 action condition，目标是像“直接遥操作视频模型”一样看到动作导致的视觉变化。

Method decomposition

Image-Layered 回答：生成视频之后怎么用

分层不是为了 Photoshop 式编辑，而是为了把视频变成 IDM、reward 和 adaptation 的训练信号。

Layer 1 Robot layer

看到臂时训 robot pose / joint estimation 和 IDM；也可辅助 wrist-view VO 结果。

Layer 2 Object-interaction

训练 process reward 和 RL 自监督；ObjectForesight 是 mesh/pose 版本的 object-layer world model 类比。

Layer 3 Scene layer

用 RoboEngine 式物理/任务感知背景替换做数据增强和 domain / self-adaptation。

Geometry VO / SLAM

看不见臂时从 wrist/head ego video 恢复 egomotion 和局部几何，再让 IDM 从 object-scene transition 推动作。

Control IDM decoder

输出 position-loop target、velocity command 或 current/torque-loop supervision。

Module Input Output Why it matters
Ego World wrist / robot-head ego views + external action multi-view action-responsive future video 回答 how should world change。
Robot layer visible arm pixels, masks, CAD/prior, joint state pose / joint estimate, IDM supervision 看得见臂时把视频和机器人状态对齐。
Object layer object mask, track, mesh/pose, contact event process reward / RL signal 回答 what interaction-centric info we need。
Scene layer background, layout, lighting, task context augmentation / adaptation scenes 在不改交互标签的前提下扩数据。

Evaluation plan

评估围绕三个问题，而不是只看视频质量

视频质量是辅助证据；主要证据是 action responsiveness、interaction state、IDM executability。

Ego World tests

  1. all views share context and synchronized action
  2. external action changes future video in the right direction
  3. robot-head and wrist views remain cross-view consistent
  4. Gamma-World / Cosmos / Wan baselines are comparable

IDM / control tests

  1. robot visible: pose / joint estimation quality
  2. robot hidden: VO / SLAM / geometry stability
  3. generated video -> IDM action smoothness
  4. position / velocity / current-loop feasibility
Metric family Primary question Why it matters
World change Does the predicted future respond correctly to the commanded action? Separates interactive world model from generic video generation.
Interaction info Do robot/object/scene layers expose the information needed for IDM and reward? Validates object-interaction-robot as a useful representation.
Executable action Can IDM convert visual transitions into feasible position, velocity, or current-loop commands? Closes the gap between video rollout and robot control.
Adaptation Can scene-layer replacement improve robustness without corrupting robot/object labels? Turns layered generation into useful data augmentation.
